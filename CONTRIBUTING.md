# Contributing to Entity Consistency Playbook

Contributions are welcome. This playbook improves when practitioners share what they learn from real entity audits.

## How to Contribute

**Suggest a new step or improvement:**
Open an issue describing the proposed change, which step it affects, and why it matters.

**Submit a correction:**
Open a pull request with the fix. If it's a factual correction, include a source.

**Add a cheatsheet or template:**
Create a new file in `cheatsheets/` or `templates/` following the format of existing files. Submit a pull request.

## Guidelines

- Keep content actionable. Each recommendation should be something a practitioner can verify or implement.
- Avoid marketing language. This is a technical resource.
- Provide evidence when possible. Link to documentation, research, or case studies.
- Write in English for global accessibility.

## Code of Conduct

Be professional and constructive. We're here to build a useful resource for the GEO community.
